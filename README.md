## 🗂️ Project Structure

```
FoodSphere/
├── 📱 /app                  # React Native / Expo Mobile App
│   ├── /screens             # Home, Menu, Cart, Checkout, Tracking, Rewards
│   ├── /components          # Shared UI components
│   ├── /store               # Redux Toolkit state management
│   └── /services            # API calls (axios)
│
├── 🖥️  /admin               # React + Vite Admin Dashboard
│   ├── /views               # SuperDashboard, Orders, Menu, Customers
│   ├── /components          # Charts, Sidebar, Notifications
│   └── /services            # api.ts (JWT auth wrapper)
│
├── 🐍 /backend              # Django REST Framework API
│   ├── /restaurants         # Menu, categories, multi-tenant config
│   ├── /orders              # Order placement & tracking
│   ├── /users               # Auth, profiles, loyalty points
│   └── /payments            # Stripe + PayFast + COD
│
└── 🌐 /websites             # 7 Individual Restaurant Sites
    ├── /seenbanao
    ├── /dineatblue
    ├── /jushhpk
    ├── /tandooristoppk
    ├── /sandmelts
    ├── /birdmanfoodspk
    └── /getafomo
```

## ✅ Features

### 📱 Mobile App
- 🏠 Home feed with all 7 restaurants
- 🛒 Cart & Checkout flow
- 📍 Live order tracking
- 🎁 Loyalty Rewards Points system
- 👤 Guest ordering (no signup required)
- 💳 Stripe + PayFast + COD payments
- 📴 Offline menu caching

### 🖥️ Admin Dashboard
- 📊 Real-time order polling & live charts
- 🔔 Firebase push notification dispatch
- 👥 Customer management & loyalty point adjustment
- 📦 Menu & category management
- 📈 30-day analytics graphs
- 📥 CSV order export (django-import-export)
- 🔐 Separate access per restaurant owner

### 🐍 Backend API
- 🏢 **Multi-tenant** — One backend for all restaurants
- 🔒 JWT Authentication (simplejwt)
- 📝 Audit logs on all admin actions
- 🧪 Automated integration test suite
- 📬 FCM push notifications
- 💰 Stripe + PayFast + COD payment logic

---

## 🏗️ Architecture Highlights

- **Multi-tenant backend** — New restaurants added via database, zero code changes
- **Unified API** — Single endpoint set, `restaurant_id` as parameter
- **Modular websites** — Each of the 7 sites deploys independently
- **Guest ordering** — JWT guest tokens, no account needed
- **Offline resilience** — App caches menu for offline viewing

## 🚀 Getting Started

### Backend
```bash
cd backend
pip install -r requirements.txt
python manage.py migrate
python manage.py runserver
```

### Admin Dashboard
```bash
cd admin
npm install
npm run dev
```

### Mobile App
```bash
cd app
npm install
npx expo start
```

### Individual Website (e.g. JushhPK)
```bash
cd websites/jushhpk
npm install
npm run dev
```
